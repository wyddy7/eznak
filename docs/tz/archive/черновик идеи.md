Есть тг бот, в нем отложенно агент будет создавать череду фраЗ, который ллм/человек отбирает фразы для челиков, планируется выкладка мб с шафлом времени выкладки + днем.


задачи: 
- проанализировать тгк это знак, его время, частоту постов, качество фраз, науччиться идентичные фразы создавать с ллм
- написать тг бота как рутер для серверной части, то есть в нем логики нет, надо фастапи передачу просто организовать
- написать бэкенд на питоне как фастапи сервер на котором вся логика будет, собрать бд в супабейз для постов + время

промпты лежат в бд тоже чтобы их можно было потом менять в боте но не приоритет

то есть архитектурно тг бот -> бэкенд -> тг бот -> тгк

воркфлоу: 
человек нажимает кнопку сгенерировать (потом по крону можно дать запуски) 
тг бот передает команду бекенду
внутри бэкенда отправляется промпт
промпт 1. Это генерирование 15 фраз на основе датасета этого знака
2. промпт про отбор
3. рефлексия и валидация и начало цикла предыдущего с предыдущим опытом (может быть сложно надо не усложнять)  
как только бэкенд решится отправить ответ человек апрувит посты Х штук. 
- Вопрос - сколько за раз? как апрувить? сделать подспорье чтобы ллм сама апрувила? 
крч человек апрувит и нажимает дальше, ему показывается когда будет каждый пост выкладываться и в какой тгк, он апрувит или сам выставляет время для каждого поста, также будет кнопка выложить прямо сейчас
как только время выставлено тг бот отправляет бд пост каждый + время поста + статус выложен не выложен в бэкэнд (или в бд напрямую?)

в намеченное время пост выкладывается.
