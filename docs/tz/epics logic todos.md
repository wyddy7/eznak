Для работы с Cursor (особенно в режиме Composer Agent / Auto) главный секрет — **не просить все сразу**. ИИ начнет путаться в файлах и ломать зависимости. Нужно вести его за ручку по слоям программы: от базы к бэкенду, от бэкенда к боту, от бота к скедулеру.

**Как использовать:**
1. Создай файл `docs/tz/tz.md` и вставь туда финальное ТЗ из предыдущего сообщения.
2. Открой окно Cursor Composer (Ctrl+I / Cmd+I, убедись что стоит режим **Agent** или **Auto**).
3. Прикрепи файл ТЗ (напиши `@tz.md` в поле ввода Composer).
4. Копируй и отправляй промпты по очереди. Жди полного завершения каждого шага (и проверяй, что код запускается).

### Промпт 3: Backend API Роуты

```text
@tz.md
Продолжаем.
Шаг 3: API Бэкенда и безопасность.
1. В `backend/api/` создай роутеры (generation.py, posts.py) и подключи их в `main.py`.

3. `POST /api/v1/generate` — вызывает llm_pipeline (для MVP channel_id — первый из БД).
4. `POST /api/v1/posts/approve` — принимает `{"phrases": [...]}`, создаёт записи в БД со статусом `draft`, возвращает `draft_posts` с `id`, `text`, `channel_id`, `suggested_time`.
5. `POST /api/v1/posts/schedule-batch` — принимает массив `[{"post_id", "channel_id", "time"}, ...]`, обновляет посты: `scheduled_at` и статус `scheduled`. Время в MSK (Europe/Moscow).
```
